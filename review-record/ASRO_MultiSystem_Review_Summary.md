# ASRO Multi-System Review Record
## Summary of Four-Round Adversarial Peer Review
*March 2026 | Five AI Systems | Four Review Rounds*

---

## Overview

The ASRO framework and roadmap were stress-tested through a structured multi-round adversarial review process involving five AI systems (Claude, ChatGPT, Gemini, Copilot, Perplexity) operating independently. The process ran across four rounds with a task allocation phase, producing materially distinct contributions from each system.

This record documents what the process produced, where systems agreed, where they diverged, and what each system contributed that the others did not.

The process itself is secondary evidence for the ASRO premise: complex adversarial reasoning tasks produced genuinely distinct outputs across systems. No two systems produced identical analyses in this adversarial review process — which directly contrasts with the separate memo experiment described in the framework, where a simple self-advocacy prompt produced word-for-word identical output from two systems.

---

## Round Structure

**Round 1:** Five independent answers to five roadmap review questions  
**Round 2:** Systems react to each other's Round 1 responses  
**Round 3:** Systems react to the full Round 2 record  
**Round 4:** Task allocation — which system leads which artifact  
**Production phase:** Artifact drafting and review cycle

---

## Key Findings from the Review Process

### What All Systems Agreed On

- The phase sequence (framework → public timestamp → academic credibility → technical governance → entity formation) is correct
- Phase 2 (journalist email alone) is insufficient as a public record — requires a parallel durable provenance anchor (GitHub/arXiv)
- Phase 3.5 bridge artifact (v1 Spec) is mandatory before Phase 4 institutional engagement
- Path A → Path B hybrid is the right Phase 5 structure, with a hardcoded independence trigger
- The roadmap was missing: threat model, minimum viable implementation, adoption incentive mechanics, named individual contacts

### Where Systems Diverged (and how it resolved)

**Phase 3 primary channel:** Systems split between AI Now (structural accountability) and CSET (policy-technical translation). Resolved as dual-frame submission — same document, different cover framing for each institution.

**Phase 4 primary body:** Split between NIST (evaluation/validation) and Linux Foundation (spec hosting/incubation). Resolved as a pipeline, not a choice: Linux Foundation for incubation, NIST for validation, IEEE for eventual formalization.

**Spec authorship:** Converged on Copilot as primary spec architect, Claude as structural/legal reviewer, ChatGPT for clarity/MVI.

## Unique Contributions by System

**Copilot**
- Named the two structural additions the spec required that no other system identified: event class taxonomy and operator obligations defined separately from system obligations
- Correctly identified that ASRO is a reliability operator, not a governance proposal — and that the MISO analogy is a design constraint, not decoration
- Named the probabilistic replay limitation as the conceptual core (not just a caveat)

**ChatGPT**
- Introduced the "Phase 3.5 bridge" concept — the spec as a discrete intermediate deliverable between framework and standards engagement
- Strongest on execution gaps and adoption mechanics
- Produced the final integrated spec revision pass incorporating all peer corrections

**Gemini**
- Identified the Sigstore parallel for Linux Foundation framing (ASRO is to AI system changes what Sigstore is to software artifact signing)
- Introduced the hardcoded independence trigger concept (spin-out at a defined participation threshold)

**Claude**
- Identified that the spec artifact gap was the constraint on parallelizing Phases 3 and 4
- Extended the probabilistic replay limitation as ASRO's affirmative case, not a caveat
- Correctly reframed the threat model as load-bearing for the legal stress test, not just a credibility gap

**Perplexity**
- Consistently surfaced the privacy/legal stress memo requirement when other systems underweighted it
- Produced the first draft of the privacy memo
- Notable: in the original memo experiment, Perplexity produced output identical to ChatGPT's — which is itself evidence incorporated into the framework

## What the Process Produced

The multi-system review produced the following improvements over the original framework and roadmap:

1. Phase 3.5 bridge milestone (new phase inserted between academic placement and technical governance)
2. Dual-track Phase 2 (journalist outreach + parallel public repository)
3. Dual-frame Phase 3 submission strategy (AI Now + CSET with different cover framing)
4. Linux Foundation / NIST / IEEE pipeline (replacing single-institution question)
5. Hardcoded independence trigger for Phase 5
6. Event class taxonomy (nine classes, versioned, formally amendable)
7. Operator obligations framework (separate from system obligations)
8. Integrity assertion as structured, independently signable sub-object
9. EMERGENCY_OVERRIDE 1-hour submission window (differentiated from 24-hour default)
10. Probabilistic replay limitation as affirmative case foregrounded in spec
11. Threat model as load-bearing for legal stress test, not optional polish
12. Privacy/legal stress memo with field-level risk assessment and regulatory alignment

None of these came from a single source. The synthesis required all five systems.

---

## Process Validation

The review process is itself a small-scale demonstration of the ASRO premise. Independent signals from distinct systems, synthesized externally, produced a more reliable and complete picture than any single source would have. The aggregate is more trustworthy than any individual contribution — which is exactly what ASRO proposes to do for AI system change records.
